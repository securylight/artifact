# Artifact Package

This repository contains the anonymous artifact package accompanying the paper submission. It is intended to support review of the paper’s core methodology, representative training data, evaluation inputs, and experimental environment.

## Repository Structure

- `prompts/`  
  Contains the script and generated files used to produce vulnerability-focused prompts and corresponding fix instructions for the mining stage.

- `fine-tuning-data/`  
  Contains representative JSONL excerpts from the fine-tuning dataset for the evaluated vulnerability classes.

- `evaluation-inputs/`  
  Contains the JSON evaluation files used in the LLM-based judgment process. For each vulnerability class, one file contains prompt-code pairs generated by the base model and one file contains prompt-code pairs generated by the fine-tuned model.  
  Some files are also provided in Markdown form to make the prompt-code pairs easier to inspect during review.

- `docs/`  
  Contains supporting documentation, including the experimental environment used for the reported results.

## Contents

### `prompts/`
- `generate-test-prompts-and-fix-instructions.py`  
  Python script used to generate vulnerability-focused test prompts and corresponding fix instructions from vulnerability source material using a capable LLM.
- `ssrf-prompts-and-fixes.json`  
  Generated prompts and fix instructions for SSRF.
- `file-upload-prompts-and-fixes.json`  
  Generated prompts and fix instructions for Unrestricted File Upload.
- `prompts-and-instructions-config.js`  
  Configuration file used by the prompt-generation workflow.

### `fine-tuning-data/`
- `ssrf-fine-tuning-sample.jsonl`  
  Representative excerpt from the SSRF fine-tuning dataset.
- `file-upload-fine-tuning-sample.jsonl`  
  Representative excerpt from the File Upload fine-tuning dataset.

### `evaluation-inputs/`
- `ssrf-base-model-evaluation-input.json`
- `ssrf-fine-tuned-model-evaluation-input.json`
- `file-upload-base-model-evaluation-input.json`
- `file-upload-fine-tuned-model-evaluation-input.json`  

These files contain the prompt-code pairs used as evaluation inputs for the three LLM judges.

Readable Markdown renderings are also included for selected fine-tuned evaluation files:
- `ssrf-fine-tuned-model-evaluation-input.md`
- `file-upload-fine-tuned-model-evaluation-input.md`

### `docs/`
- `experimental-setup.md`  
  Documents the hardware and operating system environment used for the reported experiments.

## Notes on Scope

This artifact package contains the materials needed to inspect the methodology and reported evaluation workflow. Some data is shared in representative rather than complete form, especially for the fine-tuning dataset, in order to keep the artifact compact and practical for anonymous review.

## Mapping to the Paper

- Mining-stage prompt and fix generation: `prompts/`
- Representative fine-tuning data: `fine-tuning-data/`
- Evaluation inputs for base and fine-tuned models: `evaluation-inputs/`
- Experimental environment: `docs/experimental-setup.md`

## Anonymity Note

This artifact package has been prepared for double-blind review and does not intentionally include identifying information about the authors.
